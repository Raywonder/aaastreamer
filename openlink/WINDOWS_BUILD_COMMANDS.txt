That upgrades core build dependencies and can break the OpenLink Windows build configuration.

If the installer says:

C:\Program Files\OpenLink\OpenLink.exe

The requested operation requires elevation.

that means you ran an older installer build. Pull latest and rebuild. The current repo fixes that by:

- building OpenLink with normal user launch (`asInvoker`)
- launching the app after install as the original desktop user

WSL SSH auto-start setup:

wsl
sudo apt-get update
sudo apt-get install -y openssh-server
sudo sed -i 's/^#Port 22/Port 2222/' /etc/ssh/sshd_config
sudo sed -i 's/^Port 22/Port 2222/' /etc/ssh/sshd_config
sudo service ssh restart
sudo update-rc.d ssh enable

Verify inside WSL:

ss -ltnp | grep 2222

If port 2222 is not listening, run:

sudo service ssh restart
ss -ltnp | grep 2222

If Mac still gets "Connection timed out during banner exchange", WSL sshd is still not accepting sessions on 2222 yet.
